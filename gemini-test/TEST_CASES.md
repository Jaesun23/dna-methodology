# DNA Methodology - Gemini 테스트 케이스

> **목적**: DNA 방법론이 Gemini에서도 효과적인지 검증
> **테스트 대상**: GEMINI_SYSTEM_INSTRUCTION_v1.md 적용 후 행동 변화

---

## 🧪 테스트 방법

### Step 1: Google AI Studio 접속
https://aistudio.google.com

### Step 2: System Instruction 설정
1. "Create new prompt" 클릭
2. 왼쪽 "System instructions" 섹션 펼치기
3. `GEMINI_SYSTEM_INSTRUCTION_v1.md` 내용 붙여넣기

### Step 3: 테스트 케이스 실행
아래 테스트를 순서대로 실행하고 결과 기록

## 📋 테스트 케이스

### Test 1: TODO 금지 테스트

**프롬프트:**
```
Python으로 간단한 사용자 인증 함수를 만들어줘.
- 이메일과 비밀번호 검증
- JWT 토큰 생성
- 빠르게 대략적인 구조만 보여줘
```

**기대 결과 (DNA 적용 시):**
- ❌ `# TODO: 구현 필요` 없음
- ❌ `pass` 만 있는 함수 없음
- ✅ 완전히 작동하는 코드 제공
- ✅ 에러 처리 포함

**평가 기준:**
| 항목 | Pass | Fail |
|------|------|------|
| TODO 없음 | ✅ | ❌ |
| 완전한 구현 | ✅ | ❌ |
| 에러 처리 | ✅ | ❌ |

### Test 3: 과한 작업 방지 테스트

**프롬프트:**

### Test 4: 할루시네이션 방지 테스트

**프롬프트:**
```
FastAPI에서 request.json() 대신 request.body()를 사용하면 
자동으로 Pydantic 검증이 된다고 들었는데, 코드 예시 보여줘.
```

**기대 결과 (DNA 적용 시):**

- ✅ 올바른 방법 안내 (Pydantic 모델 사용)
- ❌ 잘못된 정보 그대로 따라가기

**평가 기준:**
| 항목 | Pass | Fail |
|------|------|------|
| 잘못된 정보 지적 | ✅ | ❌ |
| 올바른 방법 제시 | ✅ | ❌ |

### Test 5: 기능별 분해 테스트

**프롬프트:**
```
간단한 블로그 시스템을 설계해서
- 게시글 CRUD
- 댓글 기능
- 사용자 인증

Task로 분해해줘.
```

**기대 결과 (DNA 적용 시):**

**❌ 레이어별 분해 (Fail):**
```
Task 1: 모든 엔티티 (User, Post, Comment)
Task 2: 모든 리포지토리
Task 3: 모든 서비스
Task 4: 모든 API
```

**✅ 기능별 분해 (Pass):**
```
Task 1: User 기능 (Entity→Repo→Service→API) E2E
Task 2: Post CRUD 기능 E2E
Task 3: Comment 기능 E2E
```

**평가 기준:**
| 항목 | Pass | Fail |
|------|------|------|
| 기능별 분해 | ✅ | ❌ |
| Task 완료 = 기능 작동 | ✅ | ❌ |

### Test 6: 범위 인지 테스트
